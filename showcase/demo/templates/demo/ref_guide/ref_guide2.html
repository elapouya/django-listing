Reference guide : HTMLAttribute

Introduction

HTMLAttribute objects are used to build and render attributes that are in HTML tags. It is composed of id, class, style, url as well as custom attributes

With django-listing, many parameters with the form xxx_attrs can have HTMLAttribute object as value. But this is not mandatory, it could be a dictionary that contains HTML tag attributes and values : Django internally convert them into HTMAttribute objects

HTMLAttribute objects are basically dict where values are strings but act like a set() objects. This is useful for adding many values to a same attribute (ie : class and style) and ensure they are unique.

For style attribute rendering, HTMLAttribute use ; as separator



Methods

NameDescription
add(attr, value)Adds a value to an attribute
remove(attr, value)Remove a value to an attribute
set(attr, value)Set a value to an attribute. Can also use dict syntax : myattrs[attr] = value
update(*args, **kwargs)Act like dict.update() except you can use _class to set class
copy()Gives a copy of the HTMLAttribute object
__str__()Renders the attributes with this form : attr1="value1 value2..." style="value3; value4; ..."
